# Менеджер товарів

## Опис проєкту

**Менеджер товарів** --- це застосунок для обліку складів і товарів, що
до них належать. Кожен склад містить набір товарів, які логічно
пов'язані між собою.

Застосунок дозволяє: - керувати складами та товарами; - переглядати
детальну інформацію; - виконувати фільтрацію та сортування; - працювати
з постійним сховищем даних.

------------------------------------------------------------------------

## Ціль
Реалізувати повнофункціональний багатошаровий застосунок із графічним
інтерфейсом, який: - підтримує повний цикл роботи з даними (CRUD); -
використовує шаблон **MVVM**; - дотримується принципів **SRP, IoC,
DI**; - працює асинхронно; - взаємодіє з Sqlite-сховищем.

------------------------------------------------------------------------

## Предметна область

### Склад (Warehouse)

-   Унікальний ідентифікатор складу
-   Назва складу
-   Місцезнаходження (enum)
-   Колекція товарів
-   Загальна вартість товарів (обчислюване поле)

### Товар (Product)

-   Унікальний ідентифікатор товару
-   Ідентифікатор складу
-   Назва товару
-   Кількість
-   Ціна за одиницю
-   Категорія (enum)
-   Опис
-   Загальна вартість (обчислюване поле)

------------------------------------------------------------------------

## Функціональні можливості

### Склади

-   Перегляд списку
-   Додавання / видалення
-   Редагування
-   Детальний перегляд
-   Фільтрація, пошук, сортування

### Товари

-   Перегляд товарів складу
-   CRUD-операції
-   Детальний перегляд
-   Фільтрація, пошук, сортування

------------------------------------------------------------------------

## Сховище

-   Реалізовано Sqlite сховище
-   Не потребує додаткових компонентів
-   Автозаповнення при першому запуску
-   Підтримка цілісності даних (cascade delete)

------------------------------------------------------------------------

## Асинхронність

-   Всі операції зі сховищем --- асинхронні
-   UI не блокується
-   Відображається індикатор завантаження

------------------------------------------------------------------------

## Архітектура

Проєкт побудований за принципом багатошарової архітектури:

### 1. UI Layer

-   WPF
-   MVVM
-   Взаємодія лише з сервісами

### 2. Service Layer

-   Бізнес-логіка
-   DTO моделі
-   Робота з репозиторіями

### 3. Data Layer

-   Репозиторії
-   Моделі
-   Робота зі сховищем
